A compact menu bar, with the detail panel when you need it.

NetStats keeps the menu bar focused on the metrics you choose. Click the status item to open a native macOS panel with hardware, memory, network, IP, location, and proxy information.

Built for people who watch both system load and network state.

The app focuses on daily-use signals instead of stuffing every possible metric into the menu bar.

Hardware overview

CPU, GPU label, memory load, used memory, cached memory, and compressed memory in a sectioned panel.

Network visibility

Upload and download speed, public IPv4, geolocation, and one-click IP copy.

Clash Verge Dev status

Shows process state, system proxy, TUN, mode, subscription, proxy group, and selected node.

Configurable menu bar

Choose which metrics deserve always-on status bar space and keep the rest in the detail view.

Native macOS feel

Built with Swift, AppKit, and SwiftUI for a menu bar app that feels at home on macOS.

Chinese and English

Switch the app interface between Chinese and English based on your workflow.

Clear privacy boundaries.

NetStats is designed around local system visibility. It avoids project-owned telemetry and documents the one external lookup it uses.

Local metrics

System metrics, network speed, and Clash Verge Dev status are read and processed locally on your Mac.

No NetStats server

The app does not upload your system or proxy status to any NetStats-owned server.

IP lookup disclosure

Public IP and geolocation lookup uses ipinfo.io to resolve the current public egress address.

Install from GitHub Releases.

The first public builds are unsigned. macOS may ask you to allow the app from System Settings after download.
